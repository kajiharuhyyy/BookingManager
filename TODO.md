件名：予約管理ミニアプリ試作のお願い

梶野くん

お疲れ様です。
業務改善の一環として、簡易的な「予約管理Webアプリ」の試作をお願いしたく連絡しました。

現場から「面談（または会議室）の予約状況を一覧で把握したい」「重複予約を防ぎたい」という要望が上がっています。
まずは最小構成でよいので、基本機能を備えたデモ用アプリを作成してください。

【対象予約（想定）】
・社内の面談予約（例：担当者と顧客の面談）
※会議室予約でもOKですが、まずは “予約枠の重複チェック” ができることを重視します。

【必須機能】
	1.	予約の一覧表示
	•	日付で絞り込み（例：/reservations?date=2025-12-14）
	•	一覧には「開始時刻・終了時刻・予約名（タイトル）・担当者（任意）・ステータス」を表示
	2.	予約の新規登録
	•	入力項目：日付、開始時刻、終了時刻、タイトル（必須）、メモ（任意）、担当者（任意）
	3.	予約の編集
	4.	予約のキャンセル（削除でも可）
	5.	重複予約の防止（最重要）
	•	同じ日付で、時間帯が重なる予約は登録できないようにする
	•	例：10:00-11:00 がある場合、10:30-11:30 はNG
	•	編集時も同様にチェックすること

【ステータス（必須）】
	•	BOOKED（予約済）
	•	CANCELLED（キャンセル）
※削除ではなくキャンセル管理できると理想。ただし学習用なのでどちらでも可。

【画面（最低限）】
	•	予約一覧画面（検索付き）
	•	新規登録画面
	•	編集画面
※UIは簡易でOKですが、デモで見せられる程度の見やすさは意識してください。

【技術要件】
	•	Java 17
	•	Spring Boot（安定版）
	•	Thymeleaf による画面実装
	•	Spring Data JPA / Hibernate
	•	H2 Database（まずはインメモリでOK）
	•	Controller / Service / Repository の責務分離（Service層に重複チェックを実装すること）
	•	テストコード作成（重要ロジックのみでOK）
	•	例：重複チェックの正常系・異常系（重複したら例外 or エラー扱い）を最低1〜2本

【納品イメージ】
	•	ローカル起動手順（READMEに記載）
	•	http://localhost:8080/reservations で動作確認できること
	•	主要な仕様（重複判定ルールなど）を README に簡単にまとめること

まずは試作版で問題ありません。完成したら一度見せてください。
不明点があれば遠慮なく質問してください。

よろしくお願いします。
